# 启动阶段：初始化论文工作区

这是整个交互式论文写作工作流的起点。

## 触发条件

- 用户提供了论文题目

不要求用户此时已经提供：

- 论文大纲
- 参考文献
- 写作要求

## 必做动作

1. 根据论文题目，在 `paper_runs/` 下创建独立文件夹
2. 创建 `00_source_materials/`
3. 创建以下固定子目录：
   - `references/`
   - `notes/`
   - `cases/`
   - `data/`
   - `images/`
4. 向用户明确这个目录已经创建完成
5. 告知用户下一阶段将进入输入准备阶段
6. 不在启动阶段要求用户放资料或填写大纲

推荐执行方式：

```bash
cd /Users/jingmac/workspce_creator/NSF_writesup/writing/academic_paper_workflow
bash ./init_paper_workspace.sh [--mode confirm|auto] "论文题目"
```

## 目的

- 让资料存放位置先存在
- 让后续所有步骤都有统一的资料入口
- 让不同论文任务互不覆盖
- 初始化应先在临时目录中完整生成，成功后再创建正式论文目录，避免失败后留下半成品目录

## 启动阶段完成后的标准提示

应明确告诉用户：

- 论文独立目录已创建
- 资料目录已创建
- `00_source_materials/reference_leads.md` 已创建，可用于填写引文信息、DOI 或链接线索
- `outline.md` 已创建
- `writing_requirements.md` 已创建
- `RUN_INFO.md` 已创建，并记录输入完成后执行模式
- `progress.md` 已创建
- 接下来进入输入准备阶段
- 放资料不是启动阶段动作，而是下一阶段动作

## 阶段边界

本动作所属阶段是“启动阶段（Phase 0）”。

启动阶段中，用户只需要：

1. 确认论文题目是否为最终版
2. 等待助手创建论文工作区
3. 选择输入完成后执行模式：`逐步确认` 或 `自动执行`

启动阶段中，用户不需要：

- 放资料
- 填写大纲
- 指定资料用途
